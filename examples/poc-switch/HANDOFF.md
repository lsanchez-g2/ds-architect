## Design identity

- **Pill-shape on track + thumb.** Same `9999px` convention as Button. The toggle is two nested pills.
- **Active-axis is layout-driven.** Off → thumb left, On → thumb right, via `justify-content` flip on the track's single flex child. Smart-animate slides between endpoints.
- **State-axis is incomplete vs Button.** Default/Disabled/Focus/Invalid only — no Hover, no Pressed, no Loading. Audit signals F5/F6/F7.
- **Invalid state shows the destructive ring at REST.** Not only on Focus. F11 — apparently intentional Apollo v2 design.
- **Disabled = container-level opacity-50.** Cross-component inconsistency with Button (which uses tinted color). F8.
- **Box-type wraps Default-type in a frame.** Same inner toggle + label/description, just with a bordered + padded outer container.
- **sm-size packs typography tighter** — label uses leading-NONE (line-height: 1), default uses leading-normal (20).

## Accessibility

- `role="switch"` (use `<button role="switch">` or equivalent).
- `aria-checked`: "true" when Active=On, "false" when Off.
- `aria-invalid="true"` on Invalid state.
- `:focus-visible` for Focus ring (3px outset drop-shadow via `effect.focus-default` or `focus-destructive`).
- Disabled: `disabled` + `aria-disabled="true"` + 50% opacity at outer container.
- Touch-target compliance: only Type=Box passes WCAG 2.5.5 (62h default, 56h sm). Type=Default cells fail (42h/36h). Recommend Type=Box for primary touch surfaces.

## Animation (SP-9)

Toggle transition (Active=Off ↔ On):

- Trigger: ON_CLICK on the outer button
- Action: CHANGE_VARIANT axis=active
- Transition: SMART_ANIMATE, ~150ms ease-out (timing inferred; Step 5 unconfirmed)
- **motionReduce policy: `skip`** — reduced-motion users see Off→On as instant variant change. No slide. Different from Button.Loading Spinner which uses `slow` (4× duration).

Together Button + Switch validate SP-9 across both motion patterns: looping vs state-transition.
