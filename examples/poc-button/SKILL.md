# Apollo v2 Button — Lossless Design-System Bundle (PoC)

Source of truth for Apollo v2 Button rendering and prototyping. Generated by `ds-architect@v3` (`lsanchez-g2/ds-architect`) as a Proof-of-Concept of the lossless-extraction contract defined in `BUNDLE_SPEC.md`.

## What this bundle contains

- **`MANIFEST.json`** — bundle header: version, source file, counts, checksums.
- **`data/tokens.json`** — 84 W3C-extended design tokens (single mode: light). Includes `$description` + `$extensions` on every semantic + component token. Resolves the `{path.to.token}` references used throughout the bundle.
- **`data/components/Button.component.json`** — 264-cell variant matrix declaration, exposedProps with per-variant/size filters, full bindings map (fill / text-color / focus-ring / radius / opacity per variant×state), slots with `swapDefaults` (Loading state replaces leftIcon with Spinner), a11y, codeConnect with shadcn-parity URLs.
- **`data/components/Button.variants-samples.json`** — 3 of 264 cells serialized as full recursive node trees (Default/Default/default + Default/Loading/default + Link/Default/default). Each carries `$bindingStatus`, captured `textDecoration`, clipsContent, `$auditFindings[]`, and a `$verificationCandidate.expectations` block.
- **`data/icons/_index.json`** — 3 Lucide icons (`circle-arrow-left`, `arrow-right`, `loader-circle`). ISC license, viewBox 24×24, `currentColor` stroke. Registry includes figmaComponentKey + usedBy cross-refs.
- **`assets/icons/*.svg`** — stock Lucide SVGs (lucide-static@1.16.0).
- **`references/`** — human-readable summaries (tokens.md, components.md, patterns.md). Progressive-disclosure friendly: load these for context without consuming the full JSON data files.
- **`verification/`** — reverse-render proof artifacts from Claude Design (`claude-design-3-cell-report.html`, 17/17 expectations matched on 2026-05-15).

## Lookup order (when rendering a Button cell)

1. **`data/components/Button.variants-samples.json`** — if the requested variant cell is among the 3 sampled, render verbatim from its node tree. The cell is the source of truth.
2. **`data/components/Button.component.json`** — if the cell is NOT in the samples file, derive its rendering from the component-level `bindings` map (fill per variant×state; text-color per variant×state; focus-ring; size table; slots; variantConstraints). Predictable across the remaining 261 cells per the patterns logged in `step3-sample-findings.md`.
3. **`data/tokens.json`** — resolve every `{path.to.token}` reference. Use the active mode (`light` is the only mode shipped in this PoC; dark mode TBD).
4. **`data/icons/_index.json`** + **`assets/icons/<name>.svg`** — resolve slot icons. Lucide stock; renderer applies color via parent `currentColor`.
5. **`verification/claude-design-3-cell-report.html`** — visual ground truth for the 3 sampled cells. Diff target.

## Hard rules

1. **Never invent values.** Every property must come from a token or a variant cell. If a property is missing, refuse rather than fabricate.
2. **Resolve `{path.to.token}` against `data/tokens.json`.** Active mode = light. If a token resolves to another `{…}` reference (alias chain), follow until reaching a leaf value.
3. **Honor `boundVariable` on every node property over inline values.** The inline value in a variant cell is the snapshot at extraction time. If a binding exists, the binding wins. Apply tokens at render time.
4. **`border-radius: 9999px` (rounded-full) is universal across all Button cells.** Every variant, every state, every size. Pill stays pill. This is the canonical fidelity test target.
5. **Loading state mandates aria-busy.** Per `Button.component.json.a11y.loadingDoublePreventionContract`, the consumer MUST emit `aria-busy="true"` + disable interaction (`disabled` or equivalent + `pointer-events: none`) on Loading-state cells.
6. **`textDecoration: "UNDERLINE"` on Link variant TEXT nodes is independent of the typography token.** Don't expect a separate `underlined-typography` token; apply text-decoration directly to the text element. (SP-3 v0.2.0.)
7. **Asymmetric padding on size=default: `padding: 8px 32px`, not `padding: 32px` all sides.** Per-side capture (paddingTop/Right/Bottom/Left). Tailwind `p-` shorthand is lossy.
8. **`slots.leftIcon` is replaced by a fixed Spinner instance when `state=Loading`.** Per `swapDefaults['state=Loading']`. `showLeftIcon` prop is suppressed in Loading cells.
9. **`variantConstraints` rule out impossible combinations.** Link variant has only 4 sizes (xs/default/sm/lg); icon-* sizes hide all slot booleans and the label.
10. **Audit findings are part of the bundle.** When a cell has `$auditFindings[]` (e.g. Link variant's typography-vs-body-text mismatch), surface them in your output rather than silently render around them. The bundle reflects source-DS reality; the renderer should not paper over it.

## Known gaps + planned work

- **261 of 264 cells** still pending full node-tree serialization. The samples file demonstrates the schema; the bindings map in `Button.component.json` covers the remainder predictably.
- **Dark mode values** not yet extracted from Figma. Single mode (light) shipped. Pending `use_figma` plugin-context read.
- **Alias chains preserved symbolically** in tokens.json (`{path.to.token}`) but `get_variable_defs` returns resolved values — Apollo v2 may have deeper aliases not visible at this MCP layer.
- **Source-DS audit findings** (not bundle bugs):
  - Finding 1 (HIGH): Button.Link typography breaks inline-in-body-text usage. Apollo v2 needs `InlineLink` component or `size=inline`.
  - Finding 2 (MEDIUM): icon-size width hardcoded `44px` instead of bound to `width/w-11`.
  - Finding 3 (LOW): Link variant Figma `documentationLink` points to shadcn `#ghost` instead of `#link`.
  - Full memo: `audit-findings-for-source.md`.
- **Steps 5, 6, 8** (graph.json + prototype.json + screenshots + full reverse-render verifier) deferred.
