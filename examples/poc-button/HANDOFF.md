## Design identity

- **Pill-shape system.** Every Button — every variant, every state, every size — uses `border-radius: 9999px`. No corner-radius variation. Identity = round.
- **High-saturation brand palette.** Violet `#5a35c0` primary; high-energy lime `#e1ff6d` secondary; classical red `#dc2626` destructive.
- **Open Sans SemiBold typography.** Type scale: 14 / 16 / 18 / 20 px. Default Button labels are 18px/semibold — larger and heavier than typical 16-base systems. Standalone CTAs read confidently; **inline-link usage breaks** because the 18/600 doesn't match 16/400 body type (see Finding 1).
- **Focus rings, not borders.** Always outset drop-shadow at 3px spread. Variant-aware: destructive variant uses tinted red ring; everything else uses neutral 50%-alpha gray.
- **Loading is a structural state.** Loading swaps the leftIcon slot with an animated Spinner, shifts fill to the hover-tint, and the consumer fulfills the doublePrevention contract.

## Accessibility

- Role: `button` (use a real `<button>` element).
- Min touch target: 44×44 px. Compliant sizes: `default`, `lg`, `icon`, `icon-lg`. Non-compliant: `xs`, `sm`, `icon-xs`, `icon-sm`, all `Link-*`.
- Focus visibility: outset drop-shadow per `effect.focus-default` (or `effect.focus-destructive` for the destructive variant).
- Loading: `aria-busy="true"` + disable interaction. Consumer MUST emit per `Button.component.json.a11y.loadingDoublePreventionContract`.
- States required: Default, Hover, Focus, Loading, Disabled, Pressed.
- Disabled state uses tinted fill (e.g. `primary-disabled` = `#ede9fe`), not opacity reduction.

## Design ↔ code drift

Apollo v2 Button maps to shadcn/ui Button across all six variants (`Default → primary`, `Secondary → secondary`, `Destructive → destructive`, `Outline → outline`, `Ghost → ghost`, `Link → link`). See `Button.component.json.codeConnect.perVariantUrls`.

**Known drift:**
- Apollo v2 emits raw `width: 44px` on icon-size cells; shadcn's equivalent would bind to a token. Finding 2.
- Link variant's `documentationLink` in Figma is wired to shadcn's `#ghost` fragment instead of `#link`. Finding 3 (caught by SP-6 codeConnect.quality verifier).

## Governance rules (for source-DS owner)

The bundle promises specific behaviors. When updating Apollo v2 Button in Figma, preserve these or bump bundle version:

1. **`border-radius: 9999px` is universal.** If a future variant needs different radius, it becomes a new variant-axis value (shape=…) — not a per-variant override.
2. **Loading state = swap leftIcon + hover-tint fill + aria-busy contract.** Don't change Loading's fill convention without updating SP-5 swapDefaults.
3. **Focus ring is drop-shadow, not border.** If a variant needs a border-style focus, it goes through `effect.focus-*` token + spec patch — not inline.
4. **textDecoration on Link is on the TEXT node.** If a future Link variant gets typography-baked underline, deprecate the existing TEXT-node decoration path explicitly.
5. **Audit findings 1–3 should be addressed in the source.** They're not blockers for the bundle but they limit fidelity for consumers.
