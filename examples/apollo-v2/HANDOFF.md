## Quick Start

### 1. Import the token CSS
```html
<!-- In your HTML head, or import in your app root CSS -->
<link rel="stylesheet" href="./design-system/tokens/tokens.css" />
```

### 2. Configure Tailwind
```js
// tailwind.config.js
const apolloTokens = require('./design-system/tokens/tailwind.tokens');

module.exports = {
  darkMode: ['class', '[data-theme="dark"]'],
  theme: {
    extend: {
      ...apolloTokens,
    },
  },
};
```

### 3. Enable dark mode
```html
<!-- Add data-theme="dark" or class="dark" to <html> -->
<html data-theme="dark">
```

### 4. Use tokens in code
```ts
import { tokens } from './design-system/tokens/tokens';
// tokens.color.primary → 'var(--color-primary)'
// tokens.spacing[4] → 'var(--spacing-4)'
```

## Typography

**Typeface:** Nunito Sans (weights 300–800)

**Scale:** xs(12) → sm(14) → base(16) → lg(18) → xl(20) → 2xl(24) → 3xl(28) → 4xl(32)

**Line heights:** `leading-normal` (20–40px depending on size) and `leading-none` (100%)

**Style pattern:** `text-{size}/leading-{height}/{weight}`

```
text-sm/leading-normal/semibold  → 14px, 20px line-height, SemiBold
text-xl/leading-normal/bold      → 20px, 28px line-height, Bold
text-2xl/leading-normal/semibold → 24px, 32px line-height, -1px tracking
```

## Design Identity

Apollo v2.1 has a **strong design POV** — a distinct personality that is evident across every component.

| Dimension | Decision | Rationale |
|---|---|---|
| **Palette** | Deep violet primary (`#5a35c0`) on white/dark-near-black | Premium, authoritative, distinct from the blue-grey SaaS default |
| **Radius** | `rounded-xl` (14px) as the dominant radius | Approachable but not playful; consistent across cards, inputs, modals |
| **Typography** | Nunito Sans, weights 300–800 | Humanist geometry — readable at small sizes, personality at large sizes |
| **Shadow** | Soft multi-stop drop shadows (Elevations page) | Depth without heaviness; system-level elevation tokens |
| **Motion** | 150–300ms, ease-out/ease-in-out only | Purposeful, never decorative; respects reduced-motion |
| **Density** | Medium — comfortable touch targets, generous line-heights | Optimized for productivity UIs with screen readers in mind |

**What to preserve:** The violet-first brand expression. Any new semantic tokens for brand, highlight, or interactive states should extend from `#5a35c0` and `#3b2185`, not introduce competing hues.

## UX Patterns

### Touch Targets (WCAG 2.5.5)
All interactive components meet the 44×44px minimum after v3 fixes:

| Component | Before | After | Method |
|---|---|---|---|
| Select Menu/Item | 32px | 40px | paddingTop/Bottom 6→10px |
| Radio/Item | 41px | 44px | paddingTop=2, paddingBottom=1 on parent |
| Checkbox row | 40px | 44px | paddingTop=2, paddingBottom=2 on parent |
| Button sm | 36px | 36px | Deferred — design convention |

> **Button sm (36px)** is intentionally below 44px. It is used in tight toolbar/inline contexts where full touch-target padding would break the surrounding layout. Always pair with a visible hit-area label.

### Motion
All transitions use `--motion-duration-*` and `--motion-ease-*` tokens. Never hardcode `ms` or `cubic-bezier` values.

```css
/* correct */
transition: background-color var(--motion-duration-150) var(--motion-ease-out);

/* wrong */
transition: background-color 150ms cubic-bezier(0, 0, 0.2, 1);
```

`prefers-reduced-motion` should suppress all transitions:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after { transition-duration: 0ms !important; }
}
```

### Hover Feedback
Every interactive surface has a visible hover state. Required for all new components:
- **Color change** (background or border)
- **Cursor: pointer** on all clickable elements
- **No hover on disabled** — `pointer-events: none` on disabled states

### Icon System
- **Source:** Inline SVG, Heroicons-style outline (stroke-width 1.5–2)
- **Sizes:** `w-6 h-6` (primary), `w-5 h-5` (feature list), `w-4 h-4` (utility)
- **Decorative icons:** always `aria-hidden="true"`
- **Meaningful icons:** always paired with a visible or `sr-only` text label

## Component Checklist

Every new component must satisfy all of the following before it is considered production-ready:

### Required states
- [ ] Default
- [ ] Hover
- [ ] Focus (keyboard-navigable, visible focus ring using `ring` token)
- [ ] Disabled (`aria-disabled`, `pointer-events: none`)
- [ ] Error / destructive (where semantically applicable)
- [ ] Loading (where applicable — `aria-busy="true"`)
- [ ] Empty / no-content (where applicable)

### Token binding
- [ ] All colors bound to Mode collection semantic tokens — no hardcoded hex
- [ ] All border radii use `radius/*` tokens — no hardcoded px
- [ ] All spacing uses Tailwind spacing scale tokens
- [ ] All transitions use `motion/*` tokens

### Accessibility
- [ ] Touch target ≥ 44×44px (or documented exception with rationale)
- [ ] WCAG AA contrast on all text/icon states (4.5:1 normal, 3:1 large)
- [ ] `aria-label` or visible label on all interactive elements
- [ ] Keyboard navigable with visible focus ring

### Documentation
- [ ] Variant × state matrix complete in Figma
- [ ] Usage notes in component description field
- [ ] TypeScript props interface (`.types.ts`)
- [ ] Storybook stories covering all variants × states (`.stories.tsx`)

## Governance Rules

1. **Never hardcode color values** — always bind to a token from the Mode collection (3. Mode)
2. **Token order:** primitive → semantic → component. Never skip levels.
3. **New components must include:** Default, Hover, Focus, Disabled, and Error states minimum
4. **Dark mode is required** — all new semantic tokens must have a dark mode value in the Mode collection
5. **Spacing uses Tailwind scale only** — no arbitrary px values outside the `spacing/*` token set
6. **Border radius uses token set** — `rounded-sm` through `rounded-full` only, no hardcoded values
7. **Motion:** any transition uses `--motion-duration-*` and `--motion-ease-*` — never raw ms/cubic values
8. **Variable scopes must be specific** — never use `ALL_SCOPES` in production; always set the narrowest correct scope
